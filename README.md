# Younes Ebrahimzadeh

Technical Writer for SaaS, Cloud & DevOps Platforms

I create documentation for software products that need to be understood quickly and used without confusion.

My work focuses on SaaS platforms, cloud systems, DevOps workflows, onboarding flows, integration setup, security and credential guidance, and product communication for technical users. I write documentation that helps people complete real tasks such as configuring a platform, connecting services, understanding console features, and following operational workflows without unnecessary back-and-forth.

Alongside writing, I also work with publishing and implementation workflows using HTML, WordPress, GitLab, and Markdown. In some projects, I use visual layouts, diagrams, and HTML-based content structures to make complex information easier to scan, follow, and apply.

Core Documentation Areas
SaaS & Platform Documentation

Documentation for software platforms, administrative consoles, and user-facing technical systems. This includes product overviews, feature explanations, workflow guidance, and documentation that helps users understand how a platform works in practice.

Cloud & DevOps Documentation

Documentation for cloud tools, infrastructure systems, Kubernetes environments, monitoring platforms, and operational processes. My focus is on explaining technical workflows in a way that supports both usability and accuracy.

API & Integration Documentation

Setup guides for APIs, third-party integrations, authentication steps, token configuration, and platform connectivity. I write this content to help users complete setup tasks without getting stuck in technical details.

Console & Onboarding Documentation

Step-by-step documentation for onboarding, console navigation, platform setup, and task execution. These documents are designed to reduce friction for new users and make system adoption easier.

Security & Credential Documentation

Documentation for access management, credential handling, security workflows, and privacy-related platform instructions. I also write user-facing policy content such as privacy policies and terms of service for applications.

Documentation UX & Visual Communication

Documentation that uses layout, hierarchy, diagrams, and visual structure to make information easier to understand. This includes HTML-based content components, workflow visuals, and presentation-style documentation pages.

Documentation Publishing & Workflow Implementation

Documentation production and publishing workflows using WordPress, GitLab, HTML, Markdown, and AI-assisted drafting and implementation processes.
